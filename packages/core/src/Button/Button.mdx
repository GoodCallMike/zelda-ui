# Button

A versatile button component with built-in accessibility features and a signature jetstream hover effect.

## Features

- **Accessibility First**: Native HTML button with proper focus management and keyboard navigation
- **Jetstream Effect**: Unique hover animation that creates a light streak across the button
- **Variant Support**: Primary and secondary styling options
- **Full HTML Support**: Extends native button props for maximum flexibility

## Best Practices

- **Use Primary** for main actions (submit, save, continue)
- **Use Secondary** for supporting actions (cancel, back, reset)
- **Always provide meaningful text** - avoid generic labels like "Click here"
- **Consider loading states** for async operations
- **Test keyboard navigation** - ensure Tab and Enter/Space work properly

## Accessibility

- ✅ **Keyboard Navigation**: Tab to focus, Enter/Space to activate
- ✅ **Screen Readers**: Proper button semantics and text content
- ✅ **Focus Management**: Visible focus rings with proper contrast
- ✅ **Disabled States**: Properly communicated to assistive technology

## Examples

```tsx
// Basic usage
<Button onClick={() => console.log('clicked')}>
  Save Changes
</Button>

// With variant
<Button variant="secondary" onClick={handleCancel}>
  Cancel
</Button>

// Disabled state
<Button disabled>
  Processing...
</Button>

// With additional props
<Button 
  type="submit" 
  className="w-full"
  aria-describedby="submit-help"
>
  Submit Form
</Button>
```
